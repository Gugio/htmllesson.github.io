Form

HTML форма, являє собою елемент керування, який застосовується для збору інформації від відвідувачів сайту. Складається з текстових полів, кнопок, списків та інших елементів. Форми передають дані на сервер.

Елементи форми:
<fieldset> - групування елементів форми
<legend> - заголовок форми
<input> - створення поля форми
<textarea> - текстове поле введення
<select> - випадаючий список
<label> - підпис до поля форми
<button> - кнопки
<form action="#">
 <fieldset>
  <legend> Заголовок форми </legend>
  <label for="name"> Ім'я* </label> <input id="name" type="text">
  <label>E-mail <input id="name" type="email"></label>
  <label for="description"> Description </label><textarea id="description">
   <label><input type="radio" name="r1" required>option1</label>
   <label><input type="radio" name="r1">option2</label>
   <label><input type="radio" name="r1">option3</label>
  <input type="submit" value="Send">
 </fieldset>
</form>
Заголовок форми


Атрибути тега <form> Значення
accept-charset="ISO-8859-1" Значення атрибуту являє собою список кодування символів, який буде використовуватись для відправлення форми.
action="action.php" Обов'язковий атрибут, який вказує на обробник до якого звертаються дані форми при їх відправленні на сервер. Являє собою файл (пр. action.php), в якому описані дії з отриманими даними. У випадку, якщо вся робота виконується на стороні клієнта сценаріями JavaScript, то для атрибута action можна задати значення "#". Також допускається відправлення даних форми на mail (приклад <form action="mailto:адрес вашей электронной почты" enctype="text/plain"></form>)
autocomplete="on | off" Відповідає за запам'ятовування введених даних у текстове поле і автодоповнення їх при введенні. on - поле не захищено та його значення можна витягувати та зберігати. off - відключає автозаповнення.
name="ім'я" Задає ім'я для форми яке використовується для доступу до елементів через сценарії.
method="get | post" Повідомляє сервер про метод запиту. get визначений для отримання інформації у адресному рядку. пара "ім'я=значення" приєднується у цьому випадку до адреси після знаку "?" та розділяється "&". post відправляє на сервер дані у запиті браузера. Це дозволяє відправити більший об'єм даних, аніж у методі get який обмежується розміром у 4Кб. По-замовчуванню - get.
novalidate Відключає перевірку у кнопці відправлення форми
enctype="application/x-www-form-urlencoded | multipart/form-data | text/plain" Визначає метод кодування даних. Використовується у випадку відправки файлу, варто задати значення атрибуту multipart/form-data. По-замовчуванню значення: application/x-www-form-urlencoded

Table

Задає таблицю на сторінці. Будь-яка таблиця складається з рядків та слотів, які задаються за допомогою тегів <tr>, <td>. Всередині контейнеру <table> допускається використання наступих елементів:

<caption> - призначений для створення заголовка до таблиці і може розміщуватися лише всередині контейнера <table>, причому відразу після відкриваючого тега.
<colgroup> - призначений для завдання ширини і стилю однієї або декількох колонок таблиці.
<col> - задає ширину і інші характеристики однієї або декількох колонок таблиці.
<thead> - для для групи рядків, які представлені у верхній частині сторінки. Допукається тільки один thead
<tbody> - для зберігання одного чи кількох рядків таблиці. Це дозволяє створювати структурні блоки.
<tfoot> - група рядків, які представлені внизу таблиці. Хоча тег <tfoot> в вихідному коді повинен бути визначений до тега <tbody>, браузери відображають його в самому низу таблиці.
<tr> - служить контейнером для створення рядка таблиці
<th> - призначений для створення однієї комірки таблиці, яка позначається як заголовна.
<td> - призначений для створення однієї комірки таблиці

Синтаксис:

<table>
<tr> <td> ... </td>
</tr>
</table>
<table>
 <caption> Заголовок таблиці </caption>
 <thead>
  <tr><th> ... </th></tr>
  <tr><td> ... </td></tr>
 </thead>
 <tfoot>
  <tr><td> ... </td></tr>
 </tfoot>
 <tbody>
  <tr><td> ... </td></tr>
 </tbody>
</table>
Атрибути <table> Значення
align="left | center | right" Вирівнювання таблиці. left - по лівому краю center - по центру right - по правому краю
background="URL" Задає фонове зображення для таблиці. На відміну від звичайного зображення, для фону не встановлюється ширина та висота, і воно відображається у натуральну величину. Якщо малюнок менший за розмір елемента, то він повторюється по по-горизонталі та вертикалі.
bgcolor="колір" Встановлює колір фону для таблиці
border="число" Встановлює товщину рамки в пікселях. По-замовчуванню = 0
bordercolor="колір" Задає колір для рамки
cellpadding="значення" Встановлює відступ між границею слоту та її вмістом. По-замовчуванню = 0, тому текст "налипає" на границі свого слоту.
cellspacing="значение" Встановлює відступи між слотами таблиці у пікселях
cols="число" Кількість колонок у таблиці. Без даного атрибуту, таблиця відобразиться тільки після того як браузер загрузить та проаналізує весь її вміст.
frame="значення" вказує браузеру як відображати рамку таблиці. Можливі значення:
void - не відображати;
border - границя навколо таблиці;
above - границя по верхньому краю;
below - по нижньому краю;
hsides - тільки горизонтальні границі;
vsides - вертикальні границі;
rhs - на правій стороні таблиці;
lhs - на лівій стороні таблиці;
height="значення", width="значення" Розмір таблиці.
bordercolor="колір" Задає колір для рамки
Атрибути <th>, <td> Значення
colspan="число" Встановлює число комірок, які повинні бути об'єднані по горизонталі.
rowspan="число" Встановлює число комірок, які повинні бути об'єднані по вертикалі.
