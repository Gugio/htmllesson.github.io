Властивість position

Властивість position визначає тип методу позиціонування для елемента (static, relative, fixed, absolute).

static

<div class="static">

static є значенням за замовчуванням. Елемент з position: static; відображається як звичайно і не буде позиціонуватися будь-яким особливим способом.

</div>

relative

<div class="relative1">

relative поводиться в точності як static але лише до тих пір, поки ви не додасте деякі додаткові властивості.

</div>
<div class="relative2">

Додавання властивостей top, right, bottom, и left змінює позицію елемента, зміщуючи його в ту чи іншу сторону щодо його початкового розташування. Інший контент НЕ буде перебудовуватися, щоб заповнити будь-який вільний простір залишений цим елементом.

</div>
<div>

наступний блок

</div>

fixed

Фіксоване позиціювання елемента щодо вікна браузера. Елемент завжди займає одне і теж місце навіть при прокручуванні сторінки, немов він приклеєний до екрану. Також як і для relative, додаткові властивості top, right, bottom, и left для даного позиціонування працюватимуть.

Фіксований елемент не залишає за собою вільний простір на сторінці, де він був розташований.

<div>

А де подівся наш друг fixed?

</div>
<div class="fixed">

А я fixed хе :)

</div>
<div>

В нього тепер своє життя

</div>

absolute

absolute поводиться як fixed за винятком того, що позиціювання відраховується щодо найближчого розташованого, позиціонованого батька а не щодо вікна браузера. Якщо елемент абсолютно позиціонується і не має батька, то він використовує тіло документа і продовжує рухатись разом з усією сторінкою при скролінгу. Пам'ятайте, що "позиціонований" елемент - це елемент з будь-яким значенням властивості position крім значення static.

<div class="relative">

Я позиційований такщо ти від мене не втечеш далеко

<div class="absolute">

А я absolute ну а шо

</div>
</div>

А цей текст вам не потрібно читати, бо це тупо муть яку я написав, щоб побачити що відбувається в психопата в мозгу після того як його вирізали ножиком для чистки крумплі

z-index

Властивість z-index визначає позицію елемента по осі z (визначає порядок накладання елемента один на другий)

z-index: 1;

Вертикальне / горизонтальне вирівнювання.

В CSS є всього кілька технік центрування елементів. Якщо їх знати, то більшість завдань вирішуються просто.

Горизонтальне вирівнювання

Для центрування inline, inline-block елементів - досить поставити батьківському блокові text-align: center;

<div class="wrap">
<span>

Test Test Test

</span>
</div>

Для центрування блоку це вже не підійде, властивість просто не подіє на блок, але на інлайнові елементи в середині дочірнього елемента подіє тому що властивість передається

Блок по горизонталі центрується margin: auto;

<div class="wrap">
<div class='inner'>
<span>

Test Test Test

</span>
</div>
</div>

Вирівнювання в тексті

Для інлайнових елементів display: inline / inline-block, включаючи картинки, властивість vertical-align центрує сам інлайн-елемент в навколишньому його тексті, inline i inline-block елементах.

baseline (за замовчуванням)
top зверху
bottom знизу
middle (по середині)
sub на рівні саб елемент
super на рівні саб елемент
text-top (верхня границя на рівні з текстоМ)
text-bottom (нижня границя на рівні з текстоМ)

  • Для вертикального вирівнювання тексту краще використовувати вертикальні відступи або властивість line-height.
  • Для абсолютно позиціонованих елементів з відомою висотою (наприклад, іконок) ідеально підійде спосіб з негативною властивістю margin-top.
  • Для більш складних випадків, коли невідома висота блоку, потрібно використовувати псевдоелемент або властивість transform.
  • Ну а якщо вам пощастило настільки, що не потрібно підтримувати старі версії браузера IE, то, звичайно, краще використовувати Flexbox.

Float

float: left | right | none | inherit;

При застосуванні цієї властивості відбувається наступне:

  1. Елемент позиціонується як звичайно, а потім виводяться з потоку документа і переміщається вліво (для left) або вправо (для right) до того, як торкнеться або батьківського кордону, або іншого елемента з float.
  2. Якщо простору по горизонталі не хватає для того, щоб помістити елемент, то він рухається вниз до тих пір, поки не влізе.
  3. Інші непозиціоновані блочні елементи без float ведуть себе так, як ніби елемента з float немає, так як він випадає з потоку.
  4. Строки (inline-елементи), напроти, "знають" про float елемент і обтікають елемент по сторонах.

Ще деталі:

  1. Елемент при наявності float отримує display: block;. (навіть якщо у нього втановлена властивість display: inline / inline-block;)
    Виключенням є деякі рідкі властивості display, подібні до inline-table та run-in.
  2. Ширина флоат-блока визначається по контенту.
  3. Вертикальний margin елементів з float елементом не зливається, в відмінності від звичайних блочних елементів.
